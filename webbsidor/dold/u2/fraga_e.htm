Två frågor om CSS och koppling mellan HTML och CSS med id och class.

  1. E1: Vad menas med kaskad i CSS? Hur fungerar det?
  2. E2: I HTML-taggarna kan man ha attribut med id och class och sedan referera till dem i CSS-koden. Vad är skillnaden, när ska man använda id och när ska man använda class?

Mitt Svar

Svaren för E1 och E2 presenteras nedan.

Fråga E1

Kaskad är ett ord som beskriver hur stilinställningarna i CSS läses in. I CSS skapar man flera stilmallar för specifika HTML componenter efter varandra, vilket skapar en "kaskad" av stilinställningar. Stilmallarna läses in en i taget i den ordning de kommer, vilket betyder att senare stilmallar kan överta (overwrite) de tidigare.

Fråga E2

Skillnaden mellan id och class är att ett id kan användas för att identifiera endast ett element, medans class kan användas för att identifiera mer än ett element. Om man t.ex. vill ha ett speciellt typ av element med en speciel stil på flera ställen på sidan så kan man göra en class för det elementet. CSS-koden för den klassen kommer då att tillämpas på alla element med den klassen.
